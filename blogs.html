1. Discuss VAR, LET,CONST in JavaScript

In JavaScript, var, let, and const have different scoping rules. var is function-scoped, meaning it is only accessible within the function it is declared in and is hoisted to the top of its scope, initialized as undefined. Conversely, let and const are block-scoped, meaning they are only accessible within the block {} they are declared in. Both let and const are hoisted but not initialized, resulting in a temporal dead zone until their declaration is encountered. Additionally, const requires an initial value and cannot be reassigned, although objects and arrays it holds can be modified. Unlike var, neither let nor const create properties on the global object when declared in the global scope..

2. What are the Use Cases of null and undefined?

In JavaScript, null and undefined are used in different contexts. undefined indicates the absence of a value, typically when a variable has been declared but not initialized, a function lacks a return statement, or an object property is non-existent. It is the default value for uninitialized variables. null, on the other hand, is an intentional assignment representing the absence of any object value and is often used to signify an empty or non-existent object. Use undefined to indicate variables that have not been assigned a value and null to represent explicitly empty values.

3. What do you mean by REST API?

A REST API (Representational State Transfer Application Programming Interface) is a set of rules and conventions for building and interacting with web services. It uses standard HTTP methods like GET, POST, PUT, DELETE to perform CRUD (Create, Read, Update, Delete) operations on resources, which are identified by URLs. REST APIs are stateless, meaning each request from a client contains all the information needed to process the request. They are widely used due to their simplicity, scalability, and compatibility with various data formats such as JSON and XML
